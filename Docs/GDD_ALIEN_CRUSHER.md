## 1. 게임 개요

### 1.1 한 줄 소개
> **"외계인이 만든 구체가 되어 인류 문명을 파괴하라."**

### 1.2 컨셉 요약

| 항목 | 내용 |
|------|------|
| 핵심 재미 | 건물·차량 파괴의 쾌감 + 성장 욕구 자극 |
| 타겟 유저 | 10~35세, 캐주얼 게이머, 짧은 세션 선호 |
| 세션 길이 | 1판 60~90초 |
| 게임 방향성 | Satisfying Destruction + Roguelite 성장 |
| 레퍼런스 | Wrecking Ball, Smash City, Rolling Sky |

### 1.3 세계관
외계 문명 **"Vrak"** 은 수천 년간 지구를 관찰해왔다.
그들은 고도로 압축된 중력 물질로 만든 **"크러셔 볼(Crusher Ball)"** 을 지구로 투하해, 인류 문명을 파괴하는 실험을 시작한다.
플레이어는 **크러셔 볼을 조종**하는 Vrak의 파괴 에이전트다.

### 1.4 제작 방향성 제약
- 아트 제작은 Unity 기본 기능(Primitive, 기본 Material, 기본 Particle System) 중심으로 진행한다.
- 대부분 프리팹은 외부 모델 의존 없이 씬 내부 조합형으로 구성한다.
- 파괴 표현은 실시간 메시 파쇄보다 **색상 변화 + 파티클 + 카메라/스케일 피드백** 중심으로 설계한다.
- 외부 에셋은 **DOTween, FEEL, GUI 에셋**만 기본 허용하며, 필요 없으면 사용하지 않는다.
- 하이러키 구조와 UI는 최소 더미라도 먼저 구축해, 제작자 의존도를 낮춘다.

## 2. 핵심 게임 루프

```
[스테이지 진입]
       |
[타이밍 게임 - 낙하 충격 결정]
       |
[인게임 - 드래그로 공 조작 / 건물 파괴]
       |
[레벨업 시 스킬 선택 (인게임 로그라이트)]
       |
[제한 시간 종료 -> 점수 집계 -> 별 획득]
       |
[아웃게임 - 파괴 점수로 스킬트리 해금 / 업그레이드]
       |
[다음 스테이지 진입] <- 루프
```

### 2.1 핵심 행동 동사
- **찍는다** (타이밍 낙하)
- **밀친다** (드래그 이동)
- **부순다** (파괴)
- **성장한다** (스킬 선택 & 해금)

## 3. 아하 모먼트 설계

> 아하 모먼트는 **"파괴의 쾌감이 극대화되는 순간"** 과 **"성장이 눈에 보이는 순간"** 두 가지로 설계한다.

### 3.1 첫 번째 — 낙하 충격 (Tutorial, 스테이지 1)
- 공이 우주에서 천천히 떨어지며 화면을 가득 채우는 연출
- 타이밍 바가 등장하고 플레이어가 **퍼펙트** 타이밍에 탭
- **"PERFECT LANDING"** 텍스트와 함께 슬로우 모션으로 건물들이 일시에 날아가는 장면
- **목표**: 처음 3초 안에 "와, 이거 재밌겠다" 반응 유발

### 3.2 두 번째 — 연쇄 파괴
- 공이 건물을 밀면 건물이 **연쇄적으로 넘어지며** 다음 건물을 파괴
- 물리 기반 도미노 이펙트
- 화면 상단에 **"CHAIN x3 / x5 / x10"** 콤보 텍스트 출력
- **목표**: 루프 초반부에 반복 플레이 욕구 자극

### 3.3 세 번째 — 형태 변환 스킬
- 스킬트리에서 **드릴 전차** 해금 후 첫 사용 시
- 공이 전차로 변형되며 지면을 뚫고 지하 구조물까지 파괴
- 변환 애니메이션 + 카메라 진동 + 강렬한 사운드 이펙트
- **목표**: 첫 형태 변환 시 "와!" 순간 제공 → 소셜 공유 유도

### 3.4 네 번째 — 풀 파괴 달성
- 맵 내 모든 건물을 파괴했을 때 **"TOTAL DESTRUCTION"** 특수 연출
- 위성 뷰로 전환되며 파괴된 도시 전경 감상
- 추가 보너스 점수 + 특수 이펙트

## 4. 인게임 시스템

### 4.1 조작 방식

| 조작 | 방식 | 설명 |
|------|------|------|
| 이동 | 드래그 | 공을 원하는 방향으로 밀기 (관성 적용) |
| 타이밍 탭 | 싱글 탭 | 낙하 타이밍 바 정지 |
| 점프 스킬 | 더블 탭 | 공을 공중으로 튀어오르게 함 (해금 후) |
| 홀드 스킬 | 길게 누르기 | 소닉 붐 / 블랙홀 등 특수 발동 |

#### 드래그 물리 설계
- **입력 방향**: 드래그 방향으로 Force 적용
- **드래그 강도**: 드래그 거리에 비례 (최대 캡 설정)
- **관성**: 공은 마찰력으로 서서히 감속 (표면 재질별 마찰 계수 다름)
- **벽/경계**: 맵 경계에서 반발력 적용 (Bounce 계수 0.6~0.8)

### 4.2 크러셔 볼 기본 스탯

| 스탯 | 기본값 | 설명 |
|------|--------|------|
| 질량 (Mass) | 10t | 건물에 가하는 피해량 연산 기준 |
| 속도 (Speed) | 5 m/s | 드래그 이동 속도 최댓값 |
| 충격력 (Impact) | 1.0 | 건물 파괴 피해 배율 |
| 반발력 (Bounce) | 0.6 | 벽/건물 충돌 후 반발 계수 |

### 4.3 파괴 물리 시스템

#### 건물 내구도 등급

| 등급 | 색상 | 내구도 | 예시 오브젝트 |
|------|------|--------|---------------|
| F (Fragile) | 흰색 | 1 hit | 가판대, 가로등, 공중전화 |
| D (Default) | 회색 | 2~3 hit | 소형 주택, 승용차 |
| C (Common) | 파란색 | 4~6 hit | 상가 건물, 버스 |
| B (Bold) | 노란색 | 7~10 hit | 오피스 빌딩, 소방차 |
| A (Armored) | 주황색 | 11~20 hit | 고층 빌딩, 탱크 |
| S (Steel) | 빨간색 | 21+ hit | 요새, 발전소, 핵시설 |

#### 파괴 연산 공식
```
실제 피해량 = (공의 속도 x 질량 x 충격력) - 건물 방어력
```
- 건물 HP가 0 이하가 되면 파괴 상태로 전환하고, 색상/발광 상태를 즉시 변경한다.
- 파괴 직전 연출은 DOTween(스케일 펀치, 짧은 회전)과 FEEL(카메라/히트 피드백)로 처리한다.
- 파괴 시 **잔해(Debris) 파티클**을 풀링 기반으로 생성한다.
- 잔해 피해량 = 원 건물 피해량의 30%
- 파괴 완료 후 짧은 지연(예: 0.6~1.0초) 뒤 렌더러/콜라이더를 비활성화한다.
- 실시간 메시 분해(Fracture)는 기본 범위에서 제외하고, 고사양 옵션으로만 검토한다.

## 5. 스킬트리 시스템

### 5.1 구조 개요

스킬트리는 **아웃게임**에서 파괴 점수(DP)로 노드를 영구 해금하고,
**인게임 레벨업 시** 해금된 노드 중 랜덤 3개를 선택지로 제시한다.

```
[아웃게임] -> DP로 노드 영구 해금
                    |
[인게임] -> 레벨업 시 -> 해금된 노드 중 3개 랜덤 제시 -> 1개 선택 (1판 한정)
```

### 5.2 스킬트리 카테고리

#### A. 능력치 강화 (Stat Enhancement)

| 스킬명 | 효과 | 최대 레벨 | 해금 비용(DP) |
|--------|------|-----------|-----------|
| 중력 코어 | 질량 +15% per Lv | 5 | 500 |
| 초속 엔진 | 이동 속도 +10% per Lv | 5 | 400 |
| 충격파 | 충격력 +20% per Lv | 5 | 600 |
| 광역 폭발 | 파괴 반경 +25% per Lv | 3 | 800 |
| 철갑 돌파 | 방어 관통력 +30% per Lv | 3 | 700 |
| 연쇄 공명 | 연쇄 파괴 시 추가 피해 +50% per Lv | 3 | 900 |

#### B. 형태 변환 (Form Transformation)

> 인게임에서 발동 시 일정 시간 동안 형태가 변경되며, 각 형태마다 고유 물리 특성을 가진다.

| 형태 | 이름 | 지속 시간 | 특성 | 쿨타임 | 해금 비용(DP) |
|------|------|-----------|------|--------|-----------|
| 기본 | 크러셔 볼 | - | 기본 물리 적용 | - | 기본 |
| 드릴 전차 | Drill Tank | 15초 | 지하 관통, 이동속도 x1.5, 강화 건물 +100% 피해 | 30초 | 2,000 |
| UFO | Vrak Saucer | 12초 | 공중 이동, 지상 오브젝트 흡입 후 투척, 지형 무시 | 40초 | 2,500 |
| 슬라임 | Slime Blob | 20초 | 형태 분열(복사 피해), 벽 통과, 독 데미지 DOT | 35초 | 2,200 |
| 플라즈마 | Plasma Core | 10초 | 이동 경로에 화염 지속 피해, 폭발 반경 x2 | 45초 | 3,000 |
| 마그네틱 | Magnet Ball | 18초 | 주변 차량·소형 오브젝트 흡착 후 집합 충격 | 30초 | 2,800 |

#### C. 특수 스킬 (Active Special Skills)

| 스킬명 | 발동 방식 | 효과 | 쿨타임 | 해금 비용(DP) |
|--------|-----------|------|--------|-----------|
| 대기권 재진입 | 더블 탭 | 공이 솟구쳤다가 낙하, 착지 시 충격파 발생 | 25초 | 1,500 |
| 소닉 붐 | 3초 홀드 | 전방 직선 충격파 방출, 건물 연쇄 폭파 | 20초 | 1,200 |
| 분열 | 기기 쉐이크 | 공이 3개로 분열, 각각 독립 이동 후 재합체 | 40초 | 3,500 |
| 중력 반전 | 더블 탭 x2 | 모든 오브젝트 일시 공중에 띄웠다가 낙하 | 60초 | 4,000 |
| 블랙홀 | 5초 홀드 | 작은 블랙홀 생성, 주변 오브젝트 흡입 파괴 | 90초 | 5,000 |

## 7. 오브젝트 & 맵 설계

### 7.1 오브젝트 분류

#### 수동 오브젝트 (파괴 대상)

| 카테고리 | 세부 오브젝트 |
|---------|-------------|
| 주거 건물 | 단독주택, 아파트, 빌라, 판자촌 |
| 상업 건물 | 편의점, 백화점, 은행, 호텔 |
| 산업 시설 | 공장, 발전소, 정유소, 항만 창고 |
| 랜드마크 | 에펠탑, 자유의 여신상, 피라미드 등 (스테이지별) |
| 인프라 | 교량, 지하철역, 공항 활주로, 댐 |
| 자연물 | 나무, 바위, 담장, 울타리 |

#### 능동 오브젝트 (재생성 주기 보유)

| 오브젝트 | 재생성 주기 | 능동 행동 |
|---------|-----------|---------|
| 승용차 | 15초 | 도로를 따라 이동 |
| 트럭 | 20초 | 느리게 이동, 충돌 시 폭발 |
| 버스 | 20초 | 정류장 순환 |
| 탱크 | 30초 | 공을 향해 포격 (방해 역할) |
| 헬리콥터 | 25초 | 공중 선회, 타격 시 추락 폭발 |
| 소방차 | 18초 | 화재 진압 (화염 스킬 방해) |
| 경찰차 | 15초 | 공을 향해 빠르게 접근 (방해) |

> **재생성 규칙**: 능동 오브젝트는 파괴 후 일정 시간 뒤 맵 외곽 스폰 포인트에서 재등장

### 7.2 월드 구성

## 8. 타이밍 게임 — Landing System

### 8.1 낙하 연출 흐름

```
[스테이지 시작]
  -> 공이 우주에서 지구를 향해 낙하하는 컷씬 (2초)
  -> 화면이 맵 탑뷰로 전환
  -> 공의 그림자가 착지 예정 지점에 표시
  -> 타이밍 바 등장 (하단 고정, 좌우 왕복)
  -> 플레이어 탭 -> 낙하 충격 판정
  -> 충격 연출 후 인게임 시작
```

### 8.2 타이밍 바 설계

```
[--------| GOOD | GREAT |PERFECT| GREAT | GOOD |--------]
  MISS                                              MISS
```

| 구간 | 색상 | 충격력 배율 | 카메라 이펙트 |
|------|------|-----------|-------------|
| MISS | 빨간색 | x0.5 | 없음 |
| GOOD | 주황색 | x1.0 | 약한 진동 |
| GREAT | 노란색 | x1.5 | 중간 진동 + 슬로우 모션 |
| PERFECT | 초록색 | x2.0 | 강한 진동 + 슬로우 모션 + 파티클 |

- 타이밍 바 이동 속도는 스테이지가 진행될수록 점차 빨라짐
- 첫 3개 스테이지는 바가 느리게 이동 (튜토리얼 배려)
- 광고 시청 시 "PERFECT 구간 2배 확장" 버프 제공

### 8.3 낙하 충격 이펙트 상세

| 등급 | 시각 이펙트 | 음향 이펙트 | 추가 효과 |
|------|--------|----------|---------|
| MISS | 작은 먼지 | 둔한 충격음 | 없음 |
| GOOD | 중간 폭발 | 폭발음 | 반경 5m 피해 |
| GREAT | 큰 폭발 + 충격파 링 | 강한 폭발음 | 반경 10m 피해 + 건물 기울어짐 |
| PERFECT | 거대 폭발 + 슬로우 + 텍스트 팝업 | 웅장한 폭발음 | 반경 20m 피해 + 잔해 비산 |

## 9. 점수 & 스타 시스템

### 9.1 점수 산정 공식

```
기본 점수    = 합산(건물 등급 x 건물 기본 포인트)
연쇄 배율    = 연속 파괴 수에 따라 x1.2 ~ x3.0
착지 배율    = 타이밍 등급에 따라 x0.5 ~ x2.0
시간 보너스  = 남은 시간(초) x 10점

최종 점수 = 기본 점수 합산 x 연쇄 배율 x 착지 배율 + 시간 보너스
```

### 9.2 건물 등급별 포인트

| 등급 | 기본 포인트 |
|------|-----------|
| F (Fragile) | 10P |
| D (Default) | 30P |
| C (Common) | 70P |
| B (Bold) | 150P |
| A (Armored) | 300P |
| S (Steel) | 600P |
| 랜드마크 | 1,000~5,000P |

### 9.3 연쇄(Chain) 배율 테이블

| 연속 파괴 수 | 배율 |
|-----------|------|
| 1개 | x1.0 |
| 2~3개 | x1.2 |
| 4~6개 | x1.5 |
| 7~10개 | x2.0 |
| 11~15개 | x2.5 |
| 16개 이상 | x3.0 |

### 9.4 스타 획득 조건

| 별 | 조건 |
|-----|------|
| 1별 | 목표 점수의 50% 달성 |
| 2별 | 목표 점수의 80% 달성 |
| 3별 | 목표 점수의 100% 달성 |
| 퍼펙트 | 목표 점수의 150% 이상 + 타이밍 PERFECT |

- 3별 최초 달성 시 추가 DP 보너스 x2
- 퍼펙트 달성 시 희귀 스킨/이펙트 보상

### 9.5 인게임 레벨업 시스템

| 항목 | 내용 |

| 경험치 | 건물 파괴 시 등급에 비례한 EXP 획득 |
| 레벨업 | EXP 바 충족 시 즉시 레벨업 |

## 13. 기술 스펙 요구사항

### 13.1 엔진 & 기술 스택

| 항목 | 선택 |
|------|------|
| 게임 엔진 | Unity (LTS 버전) |
| 렌더링 | URP (Universal Render Pipeline) |
| 물리 엔진 | Unity Physics 3D Rigidbody |
| 파괴 시스템 | Unity 기본 컴포넌트 + 색상/파티클 기반 파괴 연출 |
| 연출/트윈 | DOTween (필요 구간만 적용) |
| 피드백 시스템 | FEEL (카메라/히트/UI 피드백) |
| GUI 에셋 | UI 생산성용 GUI 에셋 (허용 범위 내) |
| 광고 SDK | AdMob (Google) / Unity Ads |
| 결제 SDK | Unity IAP |
| 분석 | Firebase Analytics + GameAnalytics |
| 클라우드 저장 | Firebase Realtime DB (스킬트리 저장) |

### 13.2 성능 목표

| 항목 | 목표 |
|------|------|
| FPS | 60fps (중사양), 30fps (저사양) |
| 초기 로딩 | 3초 이내 |
| 스테이지 로딩 | 2초 이내 |
| 메모리 사용량 | 200MB 이하 |
| 배포 용량 | 150MB 이하 |
| 지원 OS | iOS 13+, Android 8.0+ |

### 13.3 파괴 물리 최적화 전략
- 건물 상태 변화는 MaterialPropertyBlock 기반 색상 갱신으로 처리
- 잔해/먼지/충격파 파티클은 Object Pooling으로 관리
- DOTween 시퀀스는 사전 생성 후 재사용 (런타임 GC 최소화)
- 원거리 건물은 **정적 배칭(Static Batching)** 처리
- 잔해(Debris) 최대 개수 캡 설정 (30개 이상 시 오래된 잔해 제거)

### 13.4 외부 에셋 사용 정책
- 기본 허용: DOTween, FEEL, GUI 에셋
- 조건부 허용: 위 3종을 보완하는 경량 유틸(팀 승인 필수)
- 기본 비허용: 대형 파괴 라이브러리, 고용량 모델팩, 무검증 셰이더 팩
- 신규 에셋 도입 시 라이선스/용량/유지보수 영향 검토를 반드시 기록

### 13.5 FEEL/DOTween 적용 가이드
- DOTween: 이동/스케일/회전/알파 전환 등 짧고 명확한 트윈에 사용
- FEEL: 착지 판정, 연쇄 파괴, 형태 변환, 레벨업 순간의 피드백 레이어에 사용
- 동일 이벤트에 중복 연출을 과도하게 쌓지 않는다 (모바일 성능 우선)
- 목표가 불분명한 연출은 제거하고, 체감이 검증된 연출만 유지한다

## 14. 용어 정의

| 용어 | 정의 |
|------|------|
| DP (Destruction Points) | 아웃게임 영구 스킬 해금에 사용되는 파괴 점수 |
| 크레딧 | 프리미엄 재화, 인앱 결제 또는 광고 시청으로 획득 |
| 에너지 | 스테이지 입장 재화, 시간당 자동 충전 |
| 크러셔 볼 | 플레이어가 조작하는 파괴 도구 (기본 형태) |
| 형태 변환 | 크러셔 볼이 일시적으로 다른 형태로 변화하는 스킬 |
| 착지 배율 | 타이밍 게임 결과에 따른 점수 곱셈 계수 |
| 연쇄 파괴 | 건물 또는 잔해가 연속으로 다른 오브젝트를 파괴하는 것 |
